# SwiftUI 藍牙開發學習筆記

## 問題 1：如何建立藍牙掃描 App？

使用 CoreBluetooth 框架，主要步驟：
1. 建立 `CBCentralManager` 管理藍牙
2. 實作 `CBCentralManagerDelegate` 監聽狀態和發現裝置
3. 呼叫 `scanForPeripherals` 開始掃描

---

## 問題 2：改用 Combine 框架

從 `@Observable` 改成 Combine：

| 項目 | @Observable (iOS 17+) | Combine (iOS 13+) |
|------|----------------------|-------------------|
| 協議 | 無需協議 | `ObservableObject` |
| 屬性標記 | 自動追蹤 | `@Published` |
| 父視圖持有 | `@State` | `@StateObject` |
| 子視圖接收 | 直接傳遞 | `@ObservedObject` |

## 問題 4：Combine 屬性包裝器關係

### 架構圖
```
SwiftUI View
┌─────────────────────────────────┐
│ @StateObject / @ObservedObject │ ← 訂閱者
└───────────────┬─────────────────┘
                │ 訂閱 objectWillChange
                ↓
┌─────────────────────────────────┐
│ class Manager: ObservableObject │ ← 發布者
│   @Published var data = []      │
└─────────────────────────────────┘
```

### 各元件說明

| 元件 | 類型 | 用途 |
|------|------|------|
| `ObservableObject` | Protocol | 讓 class 可被觀察 |
| `@Published` | Property Wrapper | 標記要追蹤的屬性 |
| `@StateObject` | Property Wrapper | 建立並持有物件（擁有者）|
| `@ObservedObject` | Property Wrapper | 接收並觀察物件（借用者）|

### 使用規則
- `@StateObject`：第一次建立物件的地方使用
- `@ObservedObject`：從父視圖傳入時使用

## 問題 5：如何寫測試

### 關鍵技巧：依賴注入 + Protocol 抽象化

```swift
// 1. 定義 Protocol
protocol CentralManagerProtocol {
    var state: CBManagerState { get }
    func scanForPeripherals(...)
    func stopScan()
}

// 2. 讓真實物件遵循
extension CBCentralManager: CentralManagerProtocol {}

// 3. 建立 Mock
class MockCentralManager: CentralManagerProtocol {
    var state: CBManagerState = .poweredOn  // 可控制
    var scanForPeripheralsCalled = false    // 追蹤呼叫
}

// 4. 依賴注入
class BluetoothManager {
    init(centralManager: CentralManagerProtocol?) { ... }
}

// 5. 測試
@Test func startScanningShouldWork() {
    let mock = MockCentralManager(state: .poweredOn)
    let manager = BluetoothManager(centralManager: mock)

    manager.startScanning()

    #expect(manager.isScanning)
    #expect(mock.scanForPeripheralsCalled)
}
```

### 測試 @Published
```swift
var receivedValues: [Bool] = []
let cancellable = manager.$isScanning.sink { receivedValues.append($0) }

manager.startScanning()
manager.stopScanning()

#expect(receivedValues == [false, true, false])
```

## 問題 6：如何連線、讀寫資料？需要裝置文件嗎？

### 連線流程
```
掃描 → 連線 → 探索服務 → 探索特徵 → 讀/寫資料
```

### 是否需要裝置文件？

| 情況 | 需要文件？ | 說明 |
|------|-----------|------|
| 標準 BLE 服務 | 不一定 | 心率(180D)、電池(180F) 有標準 UUID |
| 自訂裝置 | **需要** | 廠商提供 Service/Characteristic UUID |
| 未知裝置 | 可探索 | 連線後列出所有服務和特徵 |

### 讀寫方法
```swift
// 讀取
manager.readValue(from: characteristic)

// 寫入字串
manager.writeString("Hello", to: characteristic)

// 寫入 HEX
manager.writeHexString("01 02 0A FF", to: characteristic)

// 開啟通知（裝置主動推送）
manager.setNotification(enabled: true, for: characteristic)
```

## 問題 7：Characteristic 是什麼？

### 比喻
把藍牙裝置想像成**自動販賣機**：
- **Peripheral（裝置）** = 整台販賣機
- **Service（服務）** = 販賣機的一個區域（飲料區）
- **Characteristic（特徵）** = 按鈕/投幣口/取物口

### 階層關係
```
Peripheral（裝置）
├── Service（服務）
│   ├── Characteristic（特徵）← 資料存取點
│   └── Characteristic
└── Service
    └── Characteristic
```

### 三種操作

| 操作 | 說明 | 比喻 |
|------|------|------|
| **Read** | 讀取資料 | 看溫度計數字 |
| **Write** | 發送指令 | 按遙控器按鈕 |
| **Notify** | 接收推送 | 手機收到通知 |

### 實際範例：心率監測器
```
心率服務 (UUID: 180D)
├── 心率測量 (2A37) - Notify - 每秒推送心率
└── 感測器位置 (2A38) - Read - 讀取位置

電池服務 (UUID: 180F)
└── 電池電量 (2A19) - Read - 讀取電量百分比
```

**簡單說：Characteristic = 你跟藍牙裝置交換資料的「窗口」**
